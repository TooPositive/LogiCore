Quick Reference Cards

Key concepts at a glance

RBAC (Role-Based Access Control)

Filter at Qdrant query level — unauthorized docs never enter the pipeline. Fail-closed: empty departments = ValueError. The LLM never sees content the user cannot access.

Circuit Breaker

3-state machine: CLOSED (normal) → OPEN (fast-fail) → HALF_OPEN (probe). Error classification excludes 4xx (caller bugs). HALF_OPEN failure = immediate re-open.

RRF (Reciprocal Rank Fusion)

Fuses ranks, not scores. Avoids scale mismatch between cosine similarity (0–1) and BM25 TF-IDF (unbounded). Single Qdrant API call via prefetch.

LangGraph

Deterministic state machine for agent orchestration. Nodes are pure functions. Edges are conditional. Checkpointing enables crash recovery and HITL interrupts.

Semantic Cache

RBAC-partitioned by cl:N|dept:X|ent:Y. Physical namespace isolation (not filter). Hit rate: 15–25% (honest with RBAC, not the 60% you see in demos).

Hash Chain

SHA-256 chain over audit entries. Same tamper evidence as blockchain at EUR 0 vs EUR 8K–15K/yr. Advisory lock prevents forking under concurrency.

Two-Tier Processing

Physical necessity for streaming: 1K msgs/sec through LLM = 500s backlog/sec. Tier 1: rule-based (EUR 0.00). Tier 2: LLM (anomalies only, <0.05%).

Fail-Closed Guard

Unknown state = deny access. Empty departments = ValueError (not empty filter). Missing clearance defaults to 1 (most restrictive). Prevents silent data leaks.

01

Corporate Brain

RAG + RBAC — Hybrid Search with Query-Level Authorization
▶ Business Problem WHY

Logistics companies have mountains of unstructured documents: HR manuals, safety protocols, customs regulations, contracts. Employees search for "towary niebezpieczne" (dangerous goods) but the document says "substancje chemiczne" (chemical substances). BM25 returns nothing.

The real problem isn't search — it's authorization. A warehouse worker searches a query that happens to match an HR termination document they shouldn't see. Post-retrieval filtering means the LLM already has the content in its context window. A prompt injection in that retrieved content could leak it.

Security Architecture Decision
RBAC is enforced at the Qdrant query level. Unauthorized documents are excluded from the search results before they ever enter the pipeline. The LLM cannot leak what it never sees.

Quantified impact: With BM25 alone, 38% of natural-language queries return wrong or no results. For a company with 200 daily searches, that's 76 unanswered questions per day — each one a support ticket or a wrong operational decision.

▶ Architecture HOW

Data Flow

User Query (e.g. "towary niebezpieczne")

Key Insight

The RBAC filter is injected into the Qdrant query itself. This is not a post-processing step. The database engine applies the filter during index traversal, so unauthorized documents are never scored, never ranked, and never returned.

▶ Components Built 8 components

1. RBAC Filter (rbac.py) — Fail-Closed Guard

Builds Qdrant filter conditions from user context. The critical design choice: empty departments = ValueError, not an empty filter.

def build_qdrant_filter(user: UserContext) -> Filter:
    if not user.departments:
        raise ValueError("Empty departments - fail closed")
    return Filter(must=[
        FieldCondition(key="department_id",
                       match=MatchAny(any=user.departments)),
        FieldCondition(key="clearance_level",
                       range=Range(lte=user.clearance_level)),
    ])
Why Fail-Closed?
MatchAny([]) has undefined behavior in Qdrant — could match everything (data leak) or nothing. Both are wrong. The guard makes the failure mode explicit and immediate.

2. Hybrid Search Retriever (retriever.py) — Strategy + DI

Uses Strategy Pattern via enum for search mode selection. embed_fn is injected as a callable — tests inject mocks, production injects Azure OpenAI.

3. Sparse Vector Encoder (sparse.py) — BM25 TF with Hash Collision Fix

Tokenizer regex preserves "ISO-9001" as single token. Hash collision fix: index_map dict sums colliding indices instead of overwriting.

# Hash collision: two tokens hash to same index
# BAD:  indices = [hash(t) for t in tokens]  # overwrites
# GOOD: index_map[hash(t)] += tf              # sums
index_map: dict[int, float] = {}
for token in tokens:
    idx = hash(token) % vocab_size
    index_map[idx] = index_map.get(idx, 0.0) + tf

4. Document Ingestion (ingestion.py)

512-char chunks with word-boundary awareness. Each Qdrant point stores both dense AND sparse vectors plus full RBAC metadata.

5. Collection Schema (collections.py) — Idempotent Setup

Creates collection if not exists. Configures dense + sparse vectors. Modifier.IDF for server-side IDF weighting. Payload indexes on department_id (KEYWORD) and clearance_level (INTEGER) for fast filtering.

6. Embedding Module (embeddings.py) — ABC + Factory + Deterministic Test Double

MockEmbedder uses SHA-256 hash expansion: same text = same vector, different text = different vector, no API calls. 4 providers: Azure, Cohere, Nomic, Mock.

class MockEmbedder(BaseEmbedder):
    async def embed(self, text: str) -> list[float]:
        h = hashlib.sha256(text.encode()).digest()
        # Expand 32 bytes to 1536 floats deterministically
        return [((b & 0xF) / 15.0) * 2 - 1 for b in expand(h, 1536)]

7. API Endpoints (search.py, ingest.py)

Path traversal protection via .resolve() + allowlist. Unknown user = 403 (not 404, to avoid leaking which users exist).

DI via Callable

Where: embed_fn parameter in HybridRetriever

Why: Decouples retriever from embedding provider. Tests inject deterministic mock, production injects Azure OpenAI.

When NOT to use: When the dependency is always the same and will never need mocking/swapping.

ABC + Factory

Where: BaseEmbedder ABC + get_embedder() factory

Why: Enforces interface contract. Factory centralizes provider selection based on settings.

When NOT to use: When you have exactly one implementation and no foreseeable alternatives. Use Protocol instead.

Deterministic Test Double (SHA-256)

Where: MockEmbedder

Why: Same text = same vector on any machine. No randomness, no API calls. 56 unit tests run without credentials.

When NOT to use: When you need to test semantic similarity (use real embeddings in live tests).

Fail-Closed Guard

Where: build_qdrant_filter() raises ValueError on empty departments

Why: Undefined behavior (MatchAny([])) could leak data or silently fail. Explicit error is always safer.

When NOT to use: When "empty" is a valid state (e.g., optional filters that should match all).

Retrieval Engineering

Chunking, Re-ranking, Query Transformation, Multi-Provider Embeddings
▶ Business Problem WHY

Phase 1 finds "similar" documents but not the RIGHT ones. Naive chunking cuts contract clauses in half — the system returns a chunk containing "The penalty for late delivery is" without the actual penalty amount. Vector similarity returns irrelevant documents at top ranks because cosine similarity doesn't understand domain relevance.

Phase 2 adds four capabilities: intelligent chunking (semantic boundaries), re-ranking (domain-aware relevance scoring), query transformation (handling vague and complex queries), and query sanitization (blocking prompt injection before it reaches the LLM).

Circuit Breaker (3-State Machine)

Where: CircuitBreakerReranker wrapping any BaseReranker

States: CLOSED (normal) → OPEN (fast-fail after N failures) → HALF_OPEN (single probe request)

Why: The 568M parameter re-ranker can fail (OOM, timeout). Without circuit breaker, every request waits for timeout. With it, failures are fast and the system falls back to NoOp reranker.

CLOSED →[failure_threshold reached]→ OPEN
OPEN   →[recovery_timeout elapsed]→ HALF_OPEN
HALF_OPEN →[success]→ CLOSED
HALF_OPEN →[failure]→ OPEN  # immediate re-open

Multi-Agent Orchestration

Customs & Finance — LangGraph Workflow with HITL Gateway
▶ Business Problem WHY

LogiCore processes 12,000 invoices per year. Each invoice takes ~3 hours of manual processing: compare rates against contracts, check compliance, flag discrepancies. That's 36,000 person-hours annually.

The real problem isn't automation — it's trust. A CFO won't accept "the AI says the invoice is wrong" without understanding the reasoning path. Financial workflows need deterministic routing where the decision path is visible and auditable.

Why LangGraph, Not CrewAI
CrewAI's role-play paradigm is non-deterministic. The CFO needs to see EXACTLY which node made which decision. LangGraph state machines provide deterministic, inspectable execution paths.

2. Pairwise Scorer — Position-Swap Agreement

Run each comparison twice with candidates in swapped positions. If the judge picks a different winner based on position alone, both scores are discarded (not averaged).

Why Discard, Not Average?
Averaging masks bias. If position A wins in order (A,B) and position B wins in order (B,A), the judge is responding to position, not quality. Averaging gives a false sense of confidence. Discarding preserves signal integrity at the cost of sample size — which is the right tradeoff.

▶ Business Problem WHY

Batch processing checks temperatures every 6 hours. Refrigeration fault at 3 AM → cargo spoiled by 9 AM → EUR 180,000 in pharmaceutical cargo lost. This happens 2–3 times per year.

The harder problem: slow drift. Temperature rises 0.3°C every 30 minutes. Threshold-only detection catches nothing until it crosses the limit — by then the cargo margin is gone. Rate-of-change detection catches this 75 minutes earlier.

Why Two-Tier Processing Is a PHYSICAL Necessity
200 trucks × 1 GPS ping/30sec = ~47K messages/day. LLM processing at 1K msgs/sec creates a 500-second backlog PER SECOND. Two-tier isn't a cost optimization — it's physically impossible to process this volume through an LLM synchronously.

Anomaly Detection Logic

class AnomalyDetector:
    def check_temperature(self, reading, setpoint, cargo_type):
        margin = 2.0 if cargo_type == "pharma" else 5.0
        # Pharma: 2C margin catches degradation 75 min earlier

        # Threshold check
        if reading.temp > setpoint + margin:
            return Alert(type="THRESHOLD_EXCEEDED")

        # Rate-of-change (catches slow drift)
        rate = self.compute_rate(reading.truck_id)
        if rate > 0.5:  # 0.5C per 30 min
            return Alert(type="DRIFT_DETECTED")

        # Z-score (statistical anomaly)
        if self.z_score(reading) > 3.0 and self.window_size >= 5:
            return Alert(type="STATISTICAL_ANOMALY")

Pattern Evolution Insight
DI via Callable appears in 5 phases because testability is a cross-cutting concern. Graceful Degradation appears in 6 phases because enterprise systems must always return SOMETHING. Strategy Pattern appears in 6 phases because configurability is how you avoid hardcoded behavior. The patterns that recur most are the ones that solve universal engineering problems, not domain-specific ones.

Patterns Glossary

Complete reference for all design patterns used
PatternWhat It IsWhen to UseWhen NOT to Use
DI via Callable Inject dependencies as function parameters (callables). Decouples implementation from interface. When a component needs to be testable with mocks or swappable across environments. When the dependency is always the same and will never need mocking.
ABC + Factory Abstract Base Class defines interface. Factory function selects concrete implementation. When implementations share code (default methods) and you control all implementations. When wrapping third-party classes (use Protocol instead).
Circuit Breaker 3-state machine (CLOSED/OPEN/HALF_OPEN) that fast-fails after repeated failures. When calling unreliable external services. Prevents cascade failures. When the downstream service is local and always available.
Fail-Closed Guard Unknown/invalid state = deny access. Raises error instead of proceeding with undefined behavior. Security-critical operations where undefined behavior could leak data. When "empty" is a valid state (e.g., optional filters).
Strategy Pattern Select algorithm at runtime via enum, config, or parameter. Implementations are interchangeable. When behavior needs to be configurable (search modes, chunking strategies, routing tiers). When there's only one algorithm and no foreseeable alternatives.
Graceful Degradation Fallback chain: primary fails → secondary → cached/static response. System always responds. When any response is better than no response (fleet monitoring, search). When wrong answers are worse than no answers (medical dosage calculations).
Deterministic Test Double Mock that produces same output for same input (SHA-256 hash expansion). No randomness, no API calls. Testing components that depend on embeddings, LLM responses, or external services. When testing semantic quality (use real APIs in live tests).
Protocol (Structural Typing) Python Protocol: any class with the right methods satisfies it. No inheritance required. When wrapping third-party classes or defining pure contracts without shared code. When implementations need shared code (default methods). Use ABC.
Namespace Isolation Physical separation (not filter) of data by access context. Structural boundary. Multi-tenant caching, RBAC-partitioned storage. Single-tenant systems where all users have identical access.
Architectural Guard Security enforcement in application code (Python) at system boundaries, not in prompts. When LLM prompts could be bypassed. Python code at graph boundaries cannot be prompt-injected. When the enforcement is purely within LLM context (though this is always risky).
Hash Chain Each entry's hash includes the previous entry's hash. Modification breaks the chain. Tamper evidence for audit logs without blockchain complexity. When tamper evidence isn't required or audit volume demands batch hashing.
Two-Tier Classification Deterministic rules (keywords) handle known patterns. LLM handles the ambiguous tail. When misclassification of known patterns has asymmetric cost (financial keywords). When categories are fluid and can't be captured by deterministic rules.
Idempotent Nodes Same input always produces same output. No side effects, no LLM non-determinism. Crash recovery workflows. Replay must produce identical results. When the node inherently requires non-deterministic behavior (creative generation).
Orthogonal Interrupt HITL concern declared at compile time, not inside node code. Node doesn't know it will be interrupted. When the approval UX might change (Slack, email, dashboard) independently of workflow logic. When interrupt conditions depend on complex node state.
