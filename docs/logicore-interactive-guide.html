▼
Business Problem
A shipping clerk opens an invoice, looks up the contract, manually compares 15 line items against negotiated rates, flags discrepancies, and routes for approval. 3 hours per invoice. 50 invoices/week = 150 hours of manual cross-referencing. One missed overcharge costs EUR 12,000.
Architecture: LangGraph Audit Workflow

Components

Key Decisions
✓
Auditor is pure math, not LLM — Rate comparison must be deterministic for crash recovery. An LLM-based auditor would produce different discrepancy amounts on retry. Idempotency is a crash-recovery prerequisite.
✓
LangGraph over CrewAI — Financial workflows need deterministic routing. CrewAI's autonomous agent delegation means you can't guarantee which agent handles which step. Auditability requires predictable control flow.
✓
SQL injection is structurally impossible — Parameterized queries via asyncpg ($1 placeholders), read-only DB role (logicore_reader). It's not "tested against" — it can't happen by construction.
✓
Compliance subgraph is cost-asymmetric — Triggered by keyword, not always. Full compliance check costs 270-1176x more. You don't run the expensive path unless the data demands it.

Assessment Rigor — Judge Bias & Drift Detection

LLM judge inflates scores 10-15%. Azure silently updates models. Dashboard masks regressions.
▼
Business Problem
Your LLM judge says quality is 4.2/5. But GPT-5-mini judging GPT-5.2 has 10-15% self-preference inflation (same family). Azure silently updates models and your quality dashboard shows "all green" while precision dropped 7% — masked by a 2% improvement in another metric.
Components
JudgeConfig (family separation) PairwiseScorer BiasDetector (3 types) HumanCalibration (Spearman) BootstrapCI ModelVersionRegistry DriftDetector PromptOptimizer
Key Decisions
✓
Family-level separation — GPT-5-mini judging GPT-5.2 = same-family bias. Use Claude to judge GPT or vice versa. Never judge your own family.
✓
Discard pairwise disagreements — When A>B and B>A in position swap, don't average. Discard. Averaging preserves noise; discarding preserves signal integrity.
✓
Per-metric drift detection — Aggregate "quality score" hides regressions. A 7% precision drop + 2% recall improvement = "stable" in aggregate but broken in production.
✓
Minimum n≥30 for bias detection — Below 30, return insufficient_data instead of a potentially misleading metric.
Bias Detection Types
Position
First-option preference in A/B comparisons
Verbosity
Longer = "better" bias
Self-Preference
10-15% within same model family
Interview Talking Point
Family-level separation — GPT-5-mini judging GPT-5.2 is same-family bias. Per-metric drift detection because aggregate scores mask regressions. Discard pairwise disagreements, don't average — preserves signal integrity.

Air-Gapped Vault — Local Inference

Swiss banking and Polish pharma can't send data to cloud APIs. One env var swaps providers.
▼
Business Problem
A Swiss bank's compliance officer says "no data leaves our network." A Polish pharma company's legal team says "patient data cannot be processed by US cloud providers." Your entire AI system is hardcoded to Azure OpenAI. Either you support local inference or you lose the contract.
Architecture
LLM_PROVIDER=azure → AzureOpenAIProvider (LangChain adapter) LLM_PROVIDER=ollama → OllamaProvider (error translation + runbooks) Same agents. Same RBAC. Same search. Same tests.
Key Decisions
✓
Protocol over ABC — Structural subtyping (duck typing). No inheritance hierarchy. A provider just needs ainvoke() and get_model_name(). That's it.
✓
Lazy imports — langchain-ollama only loads when LLM_PROVIDER=ollama. Cloud deployments don't pay the import cost of local-inference dependencies.
✓
Error translation with runbooks — Ollama returns cryptic errors. The provider translates them: "Connection refused → Is Ollama running? Try: ollama serve"
Patterns
Protocol (structural subtyping) Adapter Pattern Lazy Import Factory with env config
Interview Talking Point
One environment variable swaps from Azure to Ollama. Same agents, same RBAC, same retrieval. Cloud-vs-local is a deployment decision, not an architecture decision.

Resilience Engineering

Azure outage at 3 AM crashes fleet monitoring. EUR 180K refrigeration failure.
▼
Business Problem
Azure goes down at 3 AM. Your fleet monitoring crashes. A refrigeration unit fails on a truck carrying EUR 180,000 of pharmaceutical cargo. Nobody gets alerted. The provider returns HTTP 200 with an empty string — your system treats silence as success.
Economics
EUR 14/day unrouted → EUR 2.28/day routed -84%
Components
CircuitBreaker (state machine) RetryPolicy (exp backoff + jitter) ProviderChain (ordered fallback) ResponseQualityGate ProviderChainResponse
Key Decisions
✓
Per-provider retry, not cascade retry — Each provider burns its own retry budget, then moves to next. Cascade retry keeps bouncing between degraded providers, extending the outage.
✓
Circuit breaker excludes 4xx — A 401 (bad API key) is YOUR bug, not a provider outage. Opening the breaker on auth errors means you'd fail over to a backup provider that also can't fix your credentials.
✓
Response quality gate catches 200-OK-garbage — Empty strings, invisible Unicode, content shorter than the prompt. HTTP 200 doesn't mean "correct." It means the TCP connection worked.
✓
Routing metadata in response — fallback_used, cache_used, disclaimer. The consumer knows if this answer came from the primary or a fallback — important for confidence levels.
Circuit Breaker State Machine
CLOSED (normal) → failures hit threshold → OPEN (reject fast) OPEN → timeout expires → HALF_OPEN (probe with 1 request) HALF_OPEN → success → CLOSED | failure → OPEN
Interview Talking Point
Per-provider retry burns its own budget, then moves to next. Cascade retry keeps bouncing between degraded providers. The quality gate catches 200-OK-garbage — HTTP 200 means the TCP connection worked, not that the answer is correct.

✓ 182 tests (circuit breaker states, quality gate edge cases, fallback chains)

Regulatory Shield — EU AI Act Compliance

Article 12 says "immutable audit trail." GDPR Article 17 says "right to erasure." They contradict.
▼
Business Problem
EU AI Act Article 12 requires an immutable audit trail for high-risk AI systems. GDPR Article 17 gives individuals the right to have their data erased. An employee says "delete my data." Your audit log says "this record can never be changed." Fine either way: EUR 3.5M (7% of EUR 50M turnover).
Economics
EUR 3.5M fine risk vs. EUR 5,400/yr logging cost 648:1 ratio
Three-Layer Immutability
Layer 1: DB REVOKE → PostgreSQL prevents UPDATE/DELETE on audit_log Layer 2: Frozen Pydantic → AuditEntry model is immutable in memory Layer 3: SHA-256 Hash Chain → Each entry hashes the previous — tamper evidence
PII Vault (GDPR Resolution)
audit_log ← Immutable, REVOKE DELETE, hash chain (EU AI Act Art.12) pii_vault ← Encrypted, soft-delete capable (GDPR Art.17) Two tables. Two retention lifecycles. Conflict resolved.
Key Decisions
✓
Hash chain over blockchain — Same tamper evidence at EUR 0/yr vs EUR 8K-15K/yr. Blockchain adds network consensus overhead for a single-tenant audit log. Overkill.
✓
Advisory lock prevents hash chain forking — pg_advisory_xact_lock ensures only one writer computes the hash chain at a time. Without it, concurrent writes fork the chain.
✓
Explicit created_at parameter — Python's datetime.now() and PostgreSQL's NOW() have microsecond differences. The hash includes the timestamp, so any mismatch makes every entry appear tampered. Critical fix.
✓
Atomic checkpoint+audit writes — Same asyncpg transaction. If the checkpoint saves but the audit write fails, you have an unaudited AI decision. Zero compliance gaps.
✓
Polish PII detection — Unicode-aware regex with diacritics (ą, ć, ę, ł, ń, ó, ś, ź, ż) + 11 Polish employment keywords.
Interview Talking Point
Three-layer immutability: DB REVOKE (structural), hash chain (tamper proof), frozen Pydantic (memory). The PII vault separates concerns — immutable audit trail for EU AI Act, erasable PII for GDPR. Two tables, two lifecycles, conflict resolved.

✓ 196 tests (41 unit + 13 red-team + 12 e2e + hash chain integrity + concurrency)

Fleet Guardian — Real-Time Streaming

Batch processing checks temps every 6 hours. Slow failures take 4h to detect. EUR 180K cargo loss.
▼
Business Problem
Temperature is checked every 6 hours. A refrigeration compressor starts failing slowly — temperature rises 0.3°C per hour. After 4 hours, EUR 180,000 of pharmaceutical cargo is ruined. Batch processing would have caught it at hour 6, two hours after the damage was done. And at 1,000 messages/second, an LLM call taking 500ms creates a 500-second backlog per second of ingestion.
Economics
EUR 662/day batch → EUR 0.075/day streaming -99.9%
Two-Tier Architecture
Tier 1: Rule-based AnomalyDetector (EUR 0.00/event) → Threshold: temp > setpoint + margin → Drift: rate-of-change > 0.5°C/30min (catches slow failures!) → Z-score: > 3.0 on rolling window (n ≥ 5) → Dedup: 5-min window per truck + alert_type → Filters 99.95% of messages Tier 2: FleetGuardianAgent (LLM-powered investigation) → Only on confirmed anomalies (0.05% of messages) → Context-aware: reads truck history, fleet patterns → Decision: dispatch, monitor, or escalate
3-Tier Memory Architecture
Short
LangGraph state (within run)
Medium
Redis per-truck (30-day, 100 cap)
Long
PostgreSQL fleet_memory (patterns)
Key Decisions
✓
Two-tier is a PHYSICAL necessity — 1K msgs/sec at 500ms LLM latency = 500-second backlog per ingestion second. This isn't cost optimization. It's mathematically impossible to use LLM on every event.
✓
Rate-of-change detection is mandatory — Threshold-only misses 40% of temperature incidents. Slow drift from 2°C to 6°C over 4 hours never exceeds the threshold at any single reading but ruins EUR 207K of cargo.
✓
Pharma gets 2°C margin, not 5°C — Pharma cargo regulations require 2-8°C. A 5°C margin would alert at 13°C — cargo is already destroyed. 2°C margin catches degradation 75 minutes earlier.
✓
Graceful memory degradation — Redis down? Return empty medium-term memory. PostgreSQL down? Return empty long-term. Agent falls back to stateless investigation instead of crashing.
✓
Alert deduplication — 5-minute window per truck+alert_type. Without it, a stuck sensor generates 300 alerts/hour. Dispatcher fatigue = real alerts get ignored.
Interview Talking Point
1K temperature messages per second at 500ms LLM latency = 500-second backlog per ingestion second. Rule-based Tier 1 filters 99.95% without touching the LLM. Threshold-only misses 40% of incidents — you MUST detect rate-of-change, not just thresholds.
